Content Rules

  • All gameplay must be original and created during the development timeframe
  • Games must comply with ContentDB policies
  • Freely-licensed libraries and assets from third-parties may be used (and is encouraged!)
  • AI-generated content (code, media, etc) is not allowed, see previous item
  • Boilerplate is fine, but existing games should not be used as a base
  • Games must be able to run on the latest version of Luanti (5.10.0, if released by November)
  • Non-library modules must use the gamename_modname name convention (and everything must follow the Right to a Name policy)

Submission Process

  • Submit games to ContentDB using the Jam / Game 2024 tag
  • Games must be submitted and approved BEFORE the submission window closes (late submissions will be denied)
  • Games may be submitted for approval at any time within the approval window (Nov 1 - 23); Earlier is better
  • Groups/teams may participate (all members must be included in the Maintainers ContentDB field)
  • Participants may have multiple submissions

Scoring

Scoring is entirely community-driven. Using an interface on this website, anyone with a ContentDB account may log in to participate, allowing everyone to have a voice in the outcome.

How?

Each reviewer will be presented with a randomly sorted list of all submissions to sort (from favorite to least-favorite). The average rank of each package (after automatic penalties) determines its final score.

Rankings from members who have received 10 or more total "helpful review" points on ContentDB (or have the Editor rank or higher) will have an added 10% weight. These are sort of like "mini-judges".

Badges will also be provided to be assigned to your favorite games for 6 categories: Most Innovative, Most Fun, Best Graphics, Best Audio, Best Humor, and Best Mood. Packages with the most of a badge will be considered the winner in that category (for glory).

Things to Consider

Gameplay How intuitive and enjoyable is the gameplay? Is it a fulfilling experience? Is there a clear goal?
Innovation What makes the game unique? Does it have emergent mechanics or clever gameplay?
Content Is the game well-polished or packed with stuff to do? Is the art and sound consistent and cohesive?

Penalties

Penalties Reduces final score by 20% (additive)

No git repository Game should use proper source control
Non-free media or license Game should use completely free assets and licenses
